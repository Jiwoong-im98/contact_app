# Frontend Development Guide

## 프로젝트 개요

이 프로젝트는 연락처 관리 웹 서비스의 프론트엔드이다.

모든 화면은
PRD
화면정의서
기능정의서
구현요구사항

을 기준으로 구현한다.

명세에 없는 기능은 임의로 추가하지 않는다.

---

# 기술 스택

- HTML5
- CSS3
- JavaScript (ES6+)

---

# 개발 원칙

- SDD(Specification Driven Development)를 따른다.
- 요구사항을 먼저 확인한 후 구현한다.
- 기능보다 명세를 우선한다.
- 구현 후 테스트를 수행한다.
- 코드보다 사용자 경험을 우선한다.

---

# 프로젝트 구조

페이지별로 코드를 분리한다.

예시

/static
    css/
    js/

/templates

---

# HTML 규칙

- Semantic HTML을 사용한다.
- label을 반드시 연결한다.
- button 태그를 사용한다.
- div 남용을 피한다.

---

# CSS 규칙

- 클래스명을 사용한다.
- 인라인 스타일을 사용하지 않는다.
- 중복 스타일을 최소화한다.
- 유지보수를 고려하여 작성한다.

---

# JavaScript 규칙

- ES6+ 문법을 사용한다.
- const를 우선 사용한다.
- let은 필요한 경우만 사용한다.
- var는 사용하지 않는다.
- 함수는 하나의 역할만 수행한다.

---

# API 호출

FastAPI REST API를 fetch()로 호출한다.

async/await를 사용한다.

HTTP Status Code를 확인한다.

---

# Validation

클라이언트에서도 입력값을 검증한다.

단,

보안을 위해 서버 검증을 대체하지 않는다.

---

# 에러 처리

사용자가 이해하기 쉬운 메시지를 출력한다.

내부 에러 정보는 화면에 표시하지 않는다.

---

# 화면 구현

화면정의서를 기준으로 구현한다.

UI를 임의로 변경하지 않는다.

---

# 코드 스타일

- 의미 있는 변수명을 사용한다.
- 중복 코드를 최소화한다.
- 함수는 50줄 이하를 권장한다.
- Magic Number를 사용하지 않는다.

---

# 테스트

다음을 확인한다.

- 화면 표시
- 버튼 동작
- 입력 검증
- API 호출
- 에러 처리

---

# 문서

프론트 변경 시

- README
- 화면정의서

변경 여부를 확인한다.

---

# 금지 사항

- 명세 없는 기능 추가
- 하드코딩
- 불필요한 라이브러리 추가
- API를 추측하여 호출
