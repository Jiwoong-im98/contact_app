# Security Skill

## 목적

프로젝트의 인증, 인가, 세션 및 데이터 보호를 위한 보안 규칙을 준수한다.

---

# 인증(Authentication)

- 세션 기반 인증을 사용한다.
- 인증이 필요한 API는 반드시 세션을 확인한다.
- 인증 실패 시 401을 반환한다.

---

# 인가(Authorization)

- 자신의 데이터만 조회할 수 있다.
- 자신의 데이터만 수정할 수 있다.
- 자신의 데이터만 삭제할 수 있다.
- 다른 사용자의 데이터는 접근할 수 없다.

---

# 비밀번호

- Argon2 해시를 사용한다.
- 평문 저장을 금지한다.
- 평문 비교를 금지한다.

---

# Session

- 로그인 시 새로운 Session을 생성한다.
- 로그아웃 시 Session을 삭제한다.
- 유효하지 않은 Session은 인증 실패로 처리한다.

---

# Cookie

- Session Cookie를 통해 인증한다.
- Cookie 값을 신뢰하지 않고 Session 테이블에서 검증한다.

---

# 입력값 검증

- Pydantic Validation을 사용한다.
- SQL Injection을 방지한다.
- 필요한 입력값만 허용한다.

---

# 에러 처리

로그인 실패 시

"아이디가 존재하지 않습니다."

와

"비밀번호가 틀렸습니다."

를 구분하지 않는다.

항상 동일한 메시지를 반환한다.

---

# 데이터 보호

- 사용자 데이터는 user_id 기준으로 조회한다.
- 다른 사용자의 데이터는 절대 반환하지 않는다.

---

# 체크리스트

□ 인증 확인

□ 권한 확인

□ Session 확인

□ Cookie 확인

□ 비밀번호 해시 확인

□ 입력 검증 확인

□ 사용자 데이터 격리 확인
