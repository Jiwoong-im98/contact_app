# Documentation Skill

## 목적

프로젝트 문서를 항상 최신 상태로 유지한다.

코드와 문서가 서로 다르면 안 된다.

문서는 구현이 끝난 후 작성하는 것이 아니라,
구현과 동시에 업데이트한다.

---

# 기본 원칙

- 문서는 코드와 동일한 수준으로 중요하다.
- 코드가 변경되면 관련 문서도 함께 수정한다.
- 문서와 코드가 불일치하면 문서를 수정하거나 구현을 수정한다.
- 추측해서 문서를 작성하지 않는다.
- 현재 구현 상태를 기준으로 작성한다.

---

# 문서 작성 대상

다음 문서를 관리한다.

- README.md
- TRD
- API 문서
- ERD
- 개발 규칙
- CHANGELOG

---

# README 작성 규칙

README에는 반드시 포함한다.

- 프로젝트 소개
- 실행 방법
- 기술 스택
- 프로젝트 구조
- 주요 기능
- 실행 예시

---

# API 문서

새로운 API가 추가되면 반드시 기록한다.

포함 내용

- URL
- HTTP Method
- Request
- Response
- 상태 코드
- 예외 상황

---

# DB 문서

DB가 변경되면 다음을 수정한다.

- 테이블
- 컬럼
- FK
- UNIQUE
- INDEX

---

# 기능 문서

기능이 추가되면

- 기능 설명
- 동작 순서
- 예외 처리

를 작성한다.

---

# 변경 기록

기능 변경 시 CHANGELOG를 작성한다.

포함 내용

- 날짜
- 변경 내용
- 이유
