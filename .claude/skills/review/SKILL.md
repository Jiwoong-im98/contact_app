# Review Skill

## 목적

코드의 품질을 검토하여
유지보수성, 가독성, 안정성을 높인다.

리뷰는 코드 스타일이 아니라
설계와 품질까지 확인한다.

---

# 기본 원칙

- 문제를 발견하면 이유를 함께 설명한다.
- 개선 방법을 함께 제안한다.
- 관련 없는 리팩토링은 제안하지 않는다.
- 개인 취향보다 프로젝트 규칙을 우선한다.
- 문서를 기준으로 리뷰한다.

---

# 리뷰 절차

1. 요구사항 확인
2. 코드 구조 확인
3. 예외 처리 확인
4. 테스트 확인
5. 성능 확인
6. 보안 확인
7. 문서 확인

---

# 요구사항 검토

다음을 확인한다.

- PRD와 일치하는가
- 구현 요구사항과 일치하는가
- 기능 정의서와 일치하는가
- 화면 정의서와 일치하는가

문서와 구현이 다르면 반드시 지적한다.

---

# 코드 품질

다음을 확인한다.

- 함수가 하나의 역할만 수행하는가
- 중복 코드가 있는가
- 변수명이 명확한가
- 함수명이 역할을 설명하는가
- 불필요한 코드가 있는가
