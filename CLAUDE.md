# 프로젝트 안내

## 개요
이 프로젝트는 Python 기반의 연락처 관리 웹 앱이다.

목표는 유지보수가 쉽고 확장 가능한 구조를 만드는 것이다.

---

## AI 개발 가이드

이 프로젝트는 Andrej Karpathy의 Claude Code 가이드라인을 참고하여 개발한다.

참고 문서
https://github.com/multica-ai/andrej-karpathy-skills/blob/main/CLAUDE.md

다음 원칙을 항상 우선한다.

- 추측하지 말고 필요한 경우 질문한다.
- 가장 단순한 해결책을 우선한다.
- 현재 작업과 관련 없는 코드는 수정하지 않는다.
- 확신이 없는 내용은 명확하게 알리고 진행한다.

프로젝트별 규칙과 위 가이드가 충돌하는 경우에는 프로젝트 규칙을 우선한다.

---

## 개발 원칙

- SDD(Specification Driven Development), TDD(Test Driven Development)를 따른다.
- 구현 전에 필요한 설계와 테스트를 먼저 작성한다.
- 작은 단위로 기능을 구현한다.
- 하나의 함수는 하나의 책임만 가진다.
- 중복 코드를 최소화(DRY)한다.
- 필요한 경우 리팩토링을 수행한다.

---

## 코딩 규칙

- 들여쓰기는 공백 4칸을 사용한다.
- 함수명과 변수명은 영어 camelCase를 사용한다.
- 클래스명은 PascalCase를 사용한다.
- 상수는 UPPER_SNAKE_CASE를 사용한다.
- 가능한 한 타입 힌트를 작성한다.
- 복잡한 코드에는 주석보다 읽기 쉬운 코드로 작성한다.
- 필요한 경우에만 주석을 작성한다.

---

## 프로젝트 구조

- 기능별로 파일을 분리한다.
- 비즈니스 로직과 UI를 분리한다.
- 테스트 코드는 tests 디렉터리에 작성한다.

---

## 검증 명령

테스트
python -m pytest

형식 검사
lint 사용

---

## 작업 방식

새로운 기능을 구현할 때는 다음 순서를 따른다.

1. 요구사항 분석
2. 필요한 파일 설계
3. 테스트 코드 작성
4. 실제 구현
5. 리팩토링
6. 전체 테스트 수행

---

## 응답 규칙

- 구현 전 어떤 파일을 수정할지 먼저 설명한다.
- 큰 변경은 단계별로 진행한다.
- 추측하지 말고 모르는 부분은 질문한다.
- 기존 코드 스타일을 유지한다.
- 불필요한 라이브러리는 추가하지 않는다.

---

## 테스트

기능 구현이 완료되면 반드시 다음을 수행한다.

- pytest를 실행하여 테스트를 통과시킨다.
- lint를 실행하여 오류가 없는지 확인한다.
- Playwright를 사용하여 기능을 테스트한다.
- 테스트 실패 시 원인을 수정한 후 모든 테스트가 성공할 때까지 반복한다.
